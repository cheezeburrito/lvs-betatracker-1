💡 Tips & Features

Phase 1 — Physical Receipt
📡
Real-time scanner sync
Open scanner.html on the CipherLab gun and select a Phase 1 session. Scans stream to the desktop live feed — no pasting needed.
🖥
Live scanner feed panel
The left panel in Phase 1 shows every scan as it arrives — green dot = matched, amber = not in manifest, blue = duplicate. The right panel shows live MAWBs / HAWBs / Scanned / Missing counts.
🔴
Click "Missing" to jump to incomplete CCNs
Tap the red Missing number in the Session Stats panel — instantly filters the results list to incomplete CCNs only.
⌨️
Ctrl+Enter to process paste immediately
After pasting tracking IDs into the batch scan box, press Ctrl+Enter to process without clicking anything.
🔔
Phase 1 audio alerts (scanner gun)
Not in manifest = 3 loud beeps. Duplicate = 1 short pip. No sound = good scan matched in manifest.
Phase 2 — CBSA Check
🔗
Link a Phase 1 session
Use the Link Phase 1 session dropdown to pull the deconsolidated manifest from an existing Phase 1 session — no need to re-upload the XLSX.
🛃
Upload the RNS CSV from BorderConnect
After running BorderConnect, export the RNS report and upload it in Phase 2. The app joins CCN statuses to each tracking ID automatically.
📡
Phase 2 scanner feed
Switch the gun to Phase 2 and select a CBSA session. The desktop live feed and stats panel work the same as Phase 1 — green = Released, amber = Examination Required, cyan = Arrived/Awaiting.
🔊
Phase 2 audio alerts (scanner gun)
Released = silent (no beep). Examination Required = 4-tone alarm. Arrived / Awaiting CBSA = 3 triangle pips. Not in manifest = 3 loud beeps. Duplicate = 1 pip.
🚩
Flagged count in stats panel
The amber Flagged number in Phase 2 stats shows how many scanned parcels have a non-Released CBSA status. Use Flagged Only filter to isolate them.
Exporting
⬇
Export from the stats panel
The Export Report button in the right stats panel is always visible as you scroll. Phase 1 export is colour-coded Scanned/Missing; Phase 2 export highlights flagged CCNs.
👤
Scanned By column in the export
The exported file includes which operator scanned each tracking ID, pulled from the live Supabase scan records.
Scanner Gun (CipherLab)
📲
Install the scanner as a home screen app
On the CipherLab, open scanner.html in Chrome, tap the menu → Add to Home Screen. It installs as a standalone app — no browser UI, full screen.
🔄
Phase tab on the gun
The gun app has a Phase 1 / Phase 2 toggle at the top. Each phase shows only sessions for that phase. Switch before selecting a session.
🗑
Delete individual scans on the gun
Long-press a scan entry in the feed to delete it. A confirmation sheet appears — check "Don't ask again this session" to skip confirmations for the rest of the session.
⚠️
Session complete banner on the gun
If the desktop marks a session complete while the gun operator is still scanning, an amber banner appears on the scanner screen telling them to stop.
Navigation & General
🛃
Ready to WACM?
The blue Ready to WACM? button (bottom-right, Phase 1) opens BorderConnect in a new tab so you can submit the WACM immediately.
📋
Click any CCN card to expand
In the results list, click a CCN/MAWB row to expand it and see exactly which tracking IDs are scanned vs missing (or flagged in Phase 2).
✕
Clear paste
The ✕ Clear button next to the paste box removes all scanned IDs from the local view. Supabase scan records are not deleted.
🕐
History tab
Completed sessions appear on a calendar in the History tab. Click any day to see that day's sessions and export a report.
↩
Undo Complete
Accidentally finished a session? Each completed session in the History tab has an ↩ Undo button that moves it back to Phase 1 or Phase 2.
👥
Multiple operators simultaneously
Each device sets its own operator name. All scans sync in real-time — the live feed labels each scan by operator.
📝
Session notes
Add notes to any session — synced to all operators and included in the exported Excel file on a separate Notes sheet.
⚙
Admin panel
The Admin button (top-right) lets supervisors manage operators, customers, and view session activity across all devices.
